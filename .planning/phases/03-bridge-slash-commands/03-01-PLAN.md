  <action>
Create `lib/bridge.py` with a single public function `cmd_promote(entry_id, name_override=None)`:

1. **Imports** (all stdlib): `hashlib`, `json`, `re`, `subprocess`, `datetime.date`, `pathlib.Path`. From Prism: `.config.PRISM_HOME`, `.config.get_config`, `.index.get_entry`.

2. **Gate checks (BRG-01)**:
   - Load entry via `get_entry(entry_id)`. Print `"Entry not found: {entry_id}"` and return if None.
   - Load config via `get_config()`.
   - Check `entry.get("confidence", 0) < config.get("publish_min_confidence", 0.7)` -- print `"Gate failed: confidence {conf:.2f} < {min_conf}"` and return.
   - Check `entry.get("evidence_count", 0) < config.get("publish_min_evidence", 3)` -- print `"Gate failed: evidence {count} < {min_evidence}"` and return.
   - Check `entry.get("source") == "registry"` -- print `"Gate failed: cannot promote registry-sourced engrams"` and return.

3. **Read engram content**: Build path as `PRISM_HOME / entry.get("path", "")`. Read text. Parse frontmatter with `_parse_frontmatter(content)` that splits on `---` delimiters, parses `key: value` lines from middle section, returns `(dict, body_string)`.

4. **Generate skill name (D-03)**: Implement `_generate_skill_name(entry, frontmatter)`:
   - Start with `entry.get("trigger", "")`, strip quotes.
   - Lowercase, replace `[^a-z0-9]+` with hyphens via `re.sub`, strip leading/trailing hyphens.
   - Remove stop words: `{"the", "a", "an", "is", "are", "was", "were", "be", "to", "of", "in", "for", "on", "with", "at", "by", "and", "or", "but", "not", "it", "this", "that"}`.
   - If fewer than 2 segments, append the engram kind with `_` replaced by `-` (e.g., `"react"` becomes `"react-preference"`).
   - Truncate to first 5 segments, max 60 chars (break at hyphen boundary).
   - Ensure first char is alphabetic; if not, prepend first word of kind.
   - Validate against `^[a-z][a-z0-9]*(-[a-z0-9]+)+$`; if invalid, fallback to `f"{kind.replace('_','-')}-skill"`.
   - Use `name_override` if provided (still validate pattern).

5. **Build plugin.json (BRG-02)**: Implement `_build_plugin_json(skill_name, entry, frontmatter, body)`:
   - `name`: skill_name
   - `description`: Build with `_build_description(trigger, body, kind)` -- must be >= 50 chars and contain `TRIGGER when:`. Format: first sentence from body (or trigger text expanded), then `" TRIGGER when: "` followed by 2-3 scenario phrases derived from the engram's domain and kind. Example: `"Always use TypeScript strict mode for type safety. TRIGGER when: setting up a new TypeScript project, configuring tsconfig.json, reviewing type safety settings."`.
   - `author`: from `_git_config("user.name")` using `subprocess.run(["git", "config", "user.name"], capture_output=True, text=True, timeout=5)`, fallback `"unknown"`.
   - `repository`: from `_git_repo_name()` using `subprocess.run(["git", "remote", "get-url", "origin"], ...)`, extract `org/repo` from URL, fallback `"unknown"`.
   - `category`: map from engram kind -- `{"preference": ["architecture"], "correction": ["architecture"], "procedure": ["execution-control"], "domain_fact": ["architecture"], "tool_pattern": ["tools"], "error_recipe": ["execution-control"]}`, default `["architecture"]`.
   - `source`: `"engram"` (per D-01)
   - `commit_date`: `date.today().strftime("%d-%m-%Y")` (DD-MM-YYYY per schema pattern)
   - `source_hash`: from `_git_short_hash()` using `subprocess.run(["git", "rev-parse", "--short", "HEAD"], ...)`, return None if not in git repo.

6. **Build SKILL.md (BRG-02)**: Implement `_build_skill_md(skill_name, entry, frontmatter, body)`:
   - YAML frontmatter block with `name` and `description` (same description as plugin.json).
   - Body: `# {title from trigger}`, `*Learned from {evidence_count} observations.*`, then the engram body content as-is.

7. **Write output (BRG-03, D-01, D-02)**:
   - `output_dir = Path.cwd() / "_analysis" / "extracted_skills_codebase" / skill_name`
   - `output_dir.mkdir(parents=True, exist_ok=True)`
   - Write `plugin.json` with `json.dumps(plugin, indent=2) + "\n"`
   - Write `SKILL.md` with the generated markdown
   - Print: `"Promoted: {entry_id} -> {skill_name}"`, `"  Output: {output_dir}"`, `"\nNext: /curate-skills then /publish-skills"`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| engram content -> plugin.json | Engram trigger/body text flows into plugin.json description field and SKILL.md; could contain injection strings |
| git subprocess | Git commands run via subprocess; repo could have unusual remote URLs |
| file system writes | Promoted skills written to _analysis/ in user's project directory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | T (Tampering) | bridge.py _build_plugin_json | mitigate | json.dumps() handles escaping; no raw string interpolation into JSON |
| T-03-02 | I (Info Disclosure) | bridge.py _git_config | accept | Git user.name and remote URL are not secrets; they appear in published skills by design |
| T-03-03 | D (Denial of Service) | bridge.py subprocess calls | mitigate | All subprocess.run() calls use timeout=5 to prevent hanging on broken git |
| T-03-04 | T (Tampering) | install.sh skill copy | accept | install.sh copies from trusted local repo; no remote fetch. User controls source. |
</threat_model>
